HTML5

  • HTML stands for Hyper Text Markup Language
  • HTML is the standard markup language for creating Web pages
  • HTML describes the structure of a Web page
  • HTML consists of a series of elements
  • HTML elements tell the browser how to display the content
  • HTML elements label pieces of content such as "this is a heading", "this is a paragraph", "this is a link", etc..

CSS3

CSS (Cascading Style Sheets) is used to style and lay out web pages — for example, to alter the font, color, size, and spacing of your content, split it into multiple columns, or add animations and other decorative features. This module provides a gentle beginning to your path towards CSS mastery with the basics of how it works, what the syntax looks like, and how you can start using it to add styling to HTML.

Javascript

Javascript is a programming language that allows you to implement complex functionalities on web pages. Every time a web page does more than just sit there and display static information for you to look at—displaying timely content updates, interactive maps, animated 2D/3D graphics, scrolling video jukeboxes, or more—you can bet that JavaScript is probably involved.

CSS Concepts

CSS Wrapper Layers

  • The outer level wrappers of html, head and body are a place you can put rules that flow down to the whole page until the rule is over ridden by a more specific target.
  • Universal Selector: Applies to all elements in the document. Common use is to reset the browsers user agent where it can conflict with your design

    *{}

    * { margin: 0px; padding: 0px; box-sizing: border-box; }

  • Type Selector: Matches the element names.
  • Class Selector: Matches an element whose class attribute has a value that matches the one specified after the period symbol.

  • ID Selector: Matches an element whose ID attribute has a value that matches the one specified after the hash symbol #. symbol.

    #introduction {}

  • Child Selector: Matches an element that is a direct child of another

    li>a {} - targets any <a> elements that are children of an <li>, but not othe <a>

  • Descendant Selector: Matches an element that is a descendant of another specified element, not just a direct child of that element.

    p a {} - targets any <a> element that sit inside a <p> element, even if there are other elements nested between them.

  • Adjacent Sibling Selector: Matches an element that is the next sibling of another.

    h1+p {} - targets the first <p> element after any <h1> but not other <p> elements.

  • General Sibling Selector: Matches an element that is a sibling of another.

    h1~p {} - if you had two <p> elements that are siblings of a <h1> this rile would apply to both.

Grid

  • Grids divide the display area into a grid of rows and column to enable you to keep items aligned in columns and rows. Grid items can span multiple columns and rows.
  • display: grid;

    display is added to the selector that wraps around the outer of the grid.

  • grid-template-columns: 2fr 50% 1fr;

    defines the number of columns and the size of each one. fr is special size feature for grids, it is fractio of the remaing space not consumer by specific size settings.

  • grid-template-columns: 2fr 50% 1fr;

    defines the number of columns and the size of each one. fr is special size feature for grids, it is fractio of the remaing space not consumer by specific size settings.

  • Grid-template-rows: 25% 25% 25% 25%;

    By default rows are sized to fit within the grid height

  • Grid-template: 25% 25% 25% 25% / 2fr 50% 1fr;

    Can be used to replace the previous 2 statements. Seperate the rows specification with a / from the columns specification
